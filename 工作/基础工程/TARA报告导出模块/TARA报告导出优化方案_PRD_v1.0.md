## 一、背景与目标

### 1.1 背景

**项目背景**：  
TARA（Threat Analysis and Risk Assessment）报告是汽车网络安全开发流程中的核心交付物。系统需要导出一个包含多个 Sheet 的标准 Excel 报告，将资产识别、损害场景、威胁场景、攻击路径、风险评估、网络安全目标等全链路数据整合呈现。

**当前问题**：  
系统已跑通 TARA 报告的导出流程，但基于测试项目（ID: `749528030877024256`）验证，导出的多 Sheet Excel 文件存在大量问题：
- 数据错位（表头与数据列不对应）
- 枚举值不一致（与系统常量定义不匹配）
- Sheet 结构不合理（存在冗余 Sheet）
- 复杂关系表达错误（1:N、N:N 关系未正确展开）

逐个提 Bug 效率太低，故整合为一个完整的优化方案，供后端团队集中修复。

### 1.2 目标

- 修复所有已知的数据导出问题
- 优化 Sheet 结构，删除冗余内容
- 正确表达实体间的 1:N、N:N 关系
- 提供清晰的数据展示格式

#### 3.1.2 优化方案

**① Sheet 名称变更**
```
原：数据流图
新：系统建模
```

**② 图片清晰度优化**
- 导出图片时使用更高的 DPI（建议 300 DPI 或以上）
- 或考虑使用 SVG 格式嵌入
- 确保图片在 A4 打印时清晰可辨

**③ 右侧表格结构优化**

**当前结构（问题）:**

| 通道ID | 通道名称        | 数据流ID      | 数据流名称 | 数据流关联的数据       |
| ---- | ----------- | ---------- | ----- | -------------- |
| CH-1 | T-BOX-ESS通道 | DATAFLOW-1 | 数据流-1 | 从T-BOX传输数据到ESS |
|      |             |            |       |                |


**优化后结构:**

```
┌────────────────────────────────────────────────────────────────────────────────────┐
│                          通道与数据流对应关系表                                      │  ← 新增：合并单元格标题
├──────────┬───────────────┬────────────┬────────────┬──────────────────────────────┤
│ 通道ID   │ 通道名称       │ 数据流ID   │ 数据流名称  │ 数据流描述                    │
├──────────┼───────────────┼────────────┼────────────┼──────────────────────────────┤
│ CH-1     │ T-BOX-ESS通道  │ DATAFLOW-1 │ 数据流-1   │ 从T-BOX传输数据到ESS          │
│          │               │ DATAFLOW-2 │ 数据流-2   │ xxxxxx                       │  ← 同一通道多数据流时，通道列合并
└──────────┴───────────────┴────────────┴────────────┴──────────────────────────────┘
```

**关键变更：**
1. 在表格上方增加一个**合并单元格**，内容为：`通道与数据流对应关系表`
2. 当一个通道对应多个数据流时，通道ID 和通道名称 列进行**单元格合并**，避免重复显示
3. 将原"数据流关联的数据"改名为"数据流描述"，更准确

#### 3.6.2 当前问题详情

**当前错误数据：**

| Cybersecurity goal 网络安全目标编号 | Asset ID 资产ID | Asset Name 资产名称 | Cybersecurity goal 网络安全目标 |
|------------------------------------|-----------------|--------------------|---------------------------------|
| CS-Goal-TS-1 ❌ | COMP-2 | ESS | ESS的Authorization-可授权应免受 待定义威胁场景 的影响 |
| CS-Goal-TS-2 ❌ | COMP-2 | ESS | ESS的C-机密性应免受 待定义威胁场景的 的影响 |

**问题分析：**
1. 网络安全目标编号 `CS-Goal-TS-1` 格式不对，正确格式应为 `CS-GOAL-1`
2. 当系统中不存在用户定义的网络安全目标时，不应导出自动拼接的占位数据
3. 目标描述中的"待定义威胁场景"暴露了数据未完成状态

#### 3.6.3 优化方案

**① 编号规则修正**

```
网络安全目标编号格式：CS-GOAL-{序号}
示例：CS-GOAL-1, CS-GOAL-2, CS-GOAL-3...
```

**说明**：编号全大写，使用连字符连接，与系统中风险管理模块的网络安全目标编号保持一致。

**② 数据生成逻辑**

```
IF 存在用户定义的网络安全目标 THEN
    导出用户定义的目标数据
ELSE IF 存在已完成评估的损害场景（影响评级 ≠ 可忽略）THEN
    根据损害场景自动生成网络安全目标
    格式："{资产名称}的{网络安全属性}应免受{威胁场景描述}的影响"
ELSE
    该 Sheet 显示为空（仅保留表头）
END IF
```

       │                     │                       │
       │                     └───────────┬───────────┘
       │                                 │
       │                                 ▼
       │                          ┌─────────────┐
       │                          │  风险评估   │
       │                          │(RiskAssmt)  │

**关系说明：**
- **1 个资产** 可以有 **N 个损害场景**
- **损害场景 与 威胁场景** 是 **N : N 关系**（多对多）
  - 一个损害场景可以派生多个威胁场景
  - 一个威胁场景可以关联多个损害场景
- **1 个威胁场景** 对应 **1 棵攻击树**，攻击树自动生成 **N 条攻击路径**
- **⚠️ 风险计算只使用"最可行攻击路径"**：系统从 N 条攻击路径中选取可行性等级最高的那条，用于风险值计算
- **1 个威胁场景 × 1 个关联损害场景** → **1 条风险记录**（使用最可行攻击路径的可行性等级）
- **N 个风险记录** 可以归纳到 **1 个网络安全目标**

**TARA总表中的简化关系：**
```
威胁场景 : 最可行攻击路径 = 1 : 1 （只导出参与风险计算的那条路径）
威胁场景 : 损害场景 = 1 : N （一个威胁场景关联多个损害场景时，产生多行）
```

#### 3.7.4 行展开逻辑（核心）

**⚠️ 当前阶段目标：先保证数据正确，暂不做合并单元格优化**

**基本原则：以"威胁场景 × 损害场景"组合为最小粒度展开**

**关键规则说明：**
- 损害场景与威胁场景是 **N:N** 关系（多对多）
- 一个威胁场景对应一棵攻击树，攻击树可生成多条攻击路径
- **但系统只使用"最可行的攻击路径"参与风险值计算**
- 因此在 TARA 总表中，威胁场景与攻击路径是 **1:1** 关系（只导出最可行路径）

**展开逻辑：**

```
遍历每个威胁场景 ThreatScene:
    获取该威胁场景的最可行攻击路径 BestAttackPath  // 只取一条
    遍历该威胁场景关联的每个损害场景 DamageScene:
        输出一行数据，包含：
            - 损害场景关联的资产信息
            - 损害场景信息 + 影响评估（影响等级）
            - 威胁场景信息 + 最可行攻击路径的五维因子（可行性等级）
            - 风险值 = 影响等级 × 攻击可行性（矩阵查表）
            - 风险处置决策 + 网络安全目标
```

**示例：**

假设数据关系如下：
- 威胁场景 TS-1（威胁场景1222）关联 2 个损害场景：DS-1, DS-2
- TS-1 有 3 条攻击路径，其中 **AP-2 是最可行路径**（可行性等级：高）
- 威胁场景 TS-2（威胁场景111）关联 1 个损害场景：DS-3
- TS-2 有 2 条攻击路径，其中 **AP-1 是最可行路径**（可行性等级：低）

**展开后（每行独立，威胁场景只导出最可行攻击路径）：**

| 资产    | 损害场景ID | 影响等级 | 威胁场景ID | 威胁场景描述   | 攻击路径ID | 攻击路径描述  | 可行性等级 | 风险值 | ... |
| ----- | ------ | ---- | ------ | -------- | ------ | ------- | ----- | --- | --- |
| ESS   | DS-1   | 严重   | TS-1   | 威胁场景1222 | AP-2⭐  | 步骤A→步骤D | 高     | 5   | ... |
| ESS   | DS-2   | 中等   | TS-1   | 威胁场景1222 | AP-2⭐  | 步骤A→步骤D | 高     | 3   | ... |
| 数据流-1 | DS-3   | 可忽略  | TS-2   | 威胁场景111  | AP-1⭐  | 步骤E→步骤F | 低     | 1   | ... |


**说明：**
- 每个威胁场景只导出一条攻击路径（最可行路径，标记⭐）
- 同一威胁场景关联多个损害场景时，攻击路径信息在多行中重复显示
- 风险值根据各自损害场景的影响等级 × 统一的可行性等级计算

#### 3.7.5 特殊场景处理

**场景1：威胁场景无攻击路径（未完成攻击树）**
```
IF 威胁场景.攻击路径.count == 0 THEN
    该威胁场景不出现在 TARA 总表中
    原因：没有攻击路径就无法计算可行性等级和风险值
    系统中显示为"未完成攻击树"状态
END IF
```

**场景2：损害场景无关联威胁场景**
```
IF 损害场景.威胁场景.count == 0 THEN
    该损害场景不出现在 TARA 总表中
    仅出现在"损害场景"Sheet
END IF
```

**场景3：多个损害场景关联同一威胁场景（N:N 的体现）**
```
同一威胁场景的最可行攻击路径在多行中重复显示
每行使用各自损害场景的影响等级
风险值 = 各自影响等级 × 统一的可行性等级（矩阵查表）
```

**场景4：多条攻击路径取最可行**
```
系统从威胁场景的所有攻击路径中选取可行性等级最高的那条
可行性等级排序：高 > 中 > 低 > 极低
只有最可行路径参与导出和风险计算
```
