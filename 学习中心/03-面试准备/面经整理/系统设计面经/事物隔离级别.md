mysql中四种事物隔离级别如下：

# 1. 读未提交数据(read uncommited)
允许事物读取未被其他事物提交的变更。(脏读、不可重复读、幻读都会出现)

# 2. 读已提交数据(read committed)
只允许事物读取已经被其它事物提交的数据。(能避免脏读，但不能避免不可重复读和幻读)

# 3. 可重复读(repeatable read)
确保事物可以多次从一个字段中读取相同的值。在这个事物持续期间，禁止其他事物对这个字段进行更新。
(可以避免脏读和不可重复读，但幻读仍然存在)

# 4. 串行化(serializable)
确保事物可以从一个表中读取想用的行。在这事物持续期间，禁止其他事物对该表执行插入、更新和删除操作，所有并发问题都可避免，但性能低下。
