# 原型模式

> **核心思想**：通过**复制（克隆）**已有对象来创建新对象，而不是重新走构造流程。适用于对象创建成本高、或需要大量相似对象的场景。

## 解决的问题

当对象的**创建代价很高**（需要访问 DB、读取文件、发起网络请求等），或者需要创建**大量相似对象**时，每次 `new` + 初始化会很昂贵。原型模式通过克隆一个已有的"原型对象"来快速复制。

## 深拷贝 vs 浅拷贝

这是原型模式最核心的注意点：

| | 浅拷贝 | 深拷贝 |
|--|-------|-------|
| **基本类型字段** | 独立复制 ✅ | 独立复制 ✅ |
| **引用类型字段** | 共享同一内存 ⚠️ | 独立复制 ✅ |
| **修改副本影响原本** | 会（引用字段） | 不会 |
| **实现复杂度** | 低 | 高（需递归处理嵌套） |

> 💡 Go 中的 `=` 赋值 struct 是浅拷贝；实现深拷贝需要手动处理 slice、map、指针字段。

## Go 代码示例

### 示例一：基础克隆

```go
type Address struct {
    City    string
    Street  string
}

type User struct {
    Name    string
    Age     int
    Address *Address  // 指针：需要深拷贝
    Tags    []string  // slice：需要深拷贝
}

// Clone 实现深拷贝
func (u *User) Clone() *User {
    // 基本类型字段直接赋值
    clone := &User{
        Name: u.Name,
        Age:  u.Age,
    }
    // 指针字段：创建新对象
    if u.Address != nil {
        addrCopy := *u.Address
        clone.Address = &addrCopy
    }
    // slice 字段：make + copy
    if u.Tags != nil {
        clone.Tags = make([]string, len(u.Tags))
        copy(clone.Tags, u.Tags)
    }
    return clone
}

func main() {
    original := &User{
        Name:    "Alice",
        Age:     30,
        Address: &Address{City: "Beijing", Street: "朝阳路"},
        Tags:    []string{"admin", "vip"},
    }

    clone := original.Clone()
    clone.Name = "Bob"
    clone.Address.City = "Shanghai" // 修改副本，不影响原本

    fmt.Println(original.Name)         // Alice
    fmt.Println(original.Address.City) // Beijing（未被修改）
}
```

### 示例二：原型注册表（Prototype Registry）

当有多种原型需要按名称克隆时，使用注册表管理。

```go
type Shape interface {
    Clone() Shape
    Draw()
}

type Circle struct {
    Radius int
    Color  string
}
func (c *Circle) Clone() Shape { cp := *c; return &cp }
func (c *Circle) Draw()        { fmt.Printf("Circle r=%d color=%s\n", c.Radius, c.Color) }

type Rectangle struct {
    Width, Height int
    Color         string
}
func (r *Rectangle) Clone() Shape { cp := *r; return &cp }
func (r *Rectangle) Draw()        { fmt.Printf("Rect %dx%d color=%s\n", r.Width, r.Height, r.Color) }

// 原型注册表
type ShapeRegistry struct {
    prototypes map[string]Shape
}

func NewShapeRegistry() *ShapeRegistry {
    r := &ShapeRegistry{prototypes: make(map[string]Shape)}
    // 预先注册原型
    r.prototypes["circle"] = &Circle{Radius: 10, Color: "red"}
    r.prototypes["rect"]   = &Rectangle{Width: 20, Height: 10, Color: "blue"}
    return r
}

func (r *ShapeRegistry) Get(name string) Shape {
    p, ok := r.prototypes[name]
    if !ok {
        return nil
    }
    return p.Clone() // 每次返回克隆
}

// 使用
func main() {
    registry := NewShapeRegistry()

    s1 := registry.Get("circle") // 克隆一个圆
    s2 := registry.Get("circle") // 再克隆一个圆（独立对象）
    s1.Draw()
    s2.Draw()
}
```

### 示例三：配置对象的快速复制

```go
// 系统有一个基础配置，不同环境只需修改少数字段
type ServerConfig struct {
    Host        string
    Port        int
    MaxConns    int
    Timeout     time.Duration
    Features    []string
}

func (c *ServerConfig) Clone() *ServerConfig {
    clone := *c  // 浅拷贝值类型字段
    clone.Features = make([]string, len(c.Features))
    copy(clone.Features, c.Features)
    return &clone
}

func main() {
    base := &ServerConfig{
        Host:     "localhost",
        Port:     8080,
        MaxConns: 100,
        Timeout:  30 * time.Second,
        Features: []string{"auth", "log"},
    }

    // 生产环境：基于 base 克隆，只改少数字段
    prod := base.Clone()
    prod.Host    = "prod.example.com"
    prod.MaxConns = 1000
    prod.Features = append(prod.Features, "metrics")

    // 测试环境
    test := base.Clone()
    test.Port = 9090
}
```

## 优缺点

| | 说明 |
|--|------|
| ✅ 性能提升 | 避免重复的昂贵初始化（DB 查询、文件读取） |
| ✅ 动态生成 | 运行时可以按需克隆对象，不依赖具体类 |
| ✅ 减少子类 | 相比工厂模式，不需要为每种对象建工厂类 |
| ❌ 深拷贝复杂 | 对象有复杂嵌套时，实现正确的深拷贝有挑战 |
| ❌ 循环引用 | 对象图有循环引用时，克隆需要特别处理 |

---

## 常见应用场景

| 场景 | 说明 |
|------|------|
| **对象创建代价高** | 需要 DB 查询、文件 IO、网络请求才能初始化的对象 |
| **大量相似对象** | 游戏中大量相似的怪物/道具（位置不同，属性相近） |
| **配置变体** | 基础配置克隆后按环境微调 |
| **撤销/重做** | 保存对象快照，撤销时恢复克隆 |
| **测试 fixtures** | 单元测试中快速生成多个不同状态的测试对象 |
| **缓存预热** | 从 DB 加载原型，后续请求直接克隆，不再查 DB |
