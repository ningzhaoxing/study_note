# 构建与合并：虚拟头节点下的逐步生长

> 这个文件里有四道题。21 和 2 的结构高度相似，都是虚拟头节点 + 逐步接入节点；23 和 148 则在 21 的基础上进一步扩展，把"合并两个有序链表"当作基础操作，组合出更大的能力。

---

## 虚拟头节点的用法

这类题有一个固定结构：从已有的链表中按照某种逻辑逐个取节点，接到一条新链的末端。

```go
dummy := &ListNode{}
curr := dummy  // curr 始终指向新链的末尾
for ... {
    curr.Next = 取出的节点
    curr = curr.Next
}
return dummy.Next
```

`dummy.Next` 是新链真正的头节点。之所以用虚拟头节点而不是直接操作头节点，是因为第一个节点没有前驱，如果不加虚拟头节点，就需要单独处理"新链为空时如何设置头节点"这个特殊情况。加了虚拟头节点之后，每次接节点的逻辑都是 `curr.Next = ...`，统一了。

## [合并两个有序链表](https://leetcode.cn/problems/merge-two-sorted-lists/)（LeetCode 21）

### 问题

将两个升序链表合并成一个新的升序链表。

```
l1:  1 → 2 → 4
l2:  1 → 3 → 4
结果: 1 → 1 → 2 → 3 → 4 → 4
```

### 思路

这就是归并排序里的"合并"步骤。两个链表各自有序，每次只需要比较两个链表当前的头节点，把较小的那个接到结果链末端，然后把那条链表的指针往后移一步。

```
l1: [1] → 2 → 4     l2: [1] → 3 → 4
比较：1 == 1，取 l1 的 1（<=就取l1）

result: 1
l1: 2 → 4           l2: [1] → 3 → 4
比较：2 > 1，取 l2 的 1

result: 1 → 1
l1: [2] → 4         l2: 3 → 4
...以此类推
```

循环结束时，l1 和 l2 至多有一个非空。由于剩下的部分本身已经有序，直接接到末端就好，不需要再逐个处理。

```go
func mergeTwoLists(l1, l2 *ListNode) *ListNode {
    dummy := &ListNode{}
    curr := dummy

    for l1 != nil && l2 != nil {
        if l1.Val <= l2.Val {
            curr.Next = l1
            l1 = l1.Next
        } else {
            curr.Next = l2
            l2 = l2.Next
        }
        curr = curr.Next
    }
    // 剩下的直接接上
    if l1 != nil {
        curr.Next = l1
    } else {
        curr.Next = l2
    }
    return dummy.Next
}
```

这道题复用的是原来链表的节点，没有创建新节点。

### 递归写法

也可以递归来写：较小的头节点 + 剩余两链表归并的结果。

```go
func mergeTwoLists(l1, l2 *ListNode) *ListNode {
    if l1 == nil {
        return l2
    }
    if l2 == nil {
        return l1
    }
    if l1.Val <= l2.Val {
        l1.Next = mergeTwoLists(l1.Next, l2)
        return l1
    }
    l2.Next = mergeTwoLists(l1, l2.Next)
    return l2
}
```

递归更简洁，但链表很长时有栈溢出风险。实际用迭代更安全。

## [两数相加](https://leetcode.cn/problems/add-two-numbers/)（LeetCode 2）

### 问题

两个链表各自代表一个整数，数字**逆序**存储（个位在链表头部）。将两数相加，结果同样逆序存储返回。

```
l1:  2 → 4 → 3    （表示 342）
l2:  5 → 6 → 4    （表示 465）
结果: 7 → 0 → 8   （表示 807，即 342 + 465）
```

### 关键观察

手工做加法是从个位开始，向高位进位。而链表正好是把个位存在头部，所以从头遍历链表就是从低位到高位——顺序和加法完全一致，不需要翻转。

### 思路

每一位：`sum = l1.Val + l2.Val + carry`，结果这一位是 `sum % 10`，进位是 `sum / 10`。

三种结束情况（l1 先完、l2 先完、同时完）用 `l1 != nil || l2 != nil || carry != 0` 统一处理，不存在的节点就当 0 用。把 `carry != 0` 加进循环条件，也自动处理了最高位有进位的情况（比如 999 + 1 = 1000）。

```go
func addTwoNumbers(l1, l2 *ListNode) *ListNode {
    dummy := &ListNode{}
    curr := dummy
    carry := 0

    for l1 != nil || l2 != nil || carry != 0 {
        sum := carry

        if l1 != nil {
            sum += l1.Val
            l1 = l1.Next
        }
        if l2 != nil {
            sum += l2.Val
            l2 = l2.Next
        }

        carry = sum / 10
        curr.Next = &ListNode{Val: sum % 10} // 创建新节点，不复用原链表的
        curr = curr.Next
    }
    return dummy.Next
}
```

和合并有序链表不同，这道题每一步都在创建新节点，而不是从原链表里摘节点。

---

## 两题的共同结构

```go
dummy := &ListNode{}
curr := dummy
for 还没处理完 {
    curr.Next = 下一个要接的节点
    curr = curr.Next
}
return dummy.Next
```

这个结构在链表题里很常见，本质是"用 curr 指针不断延伸一条新链"。虚拟头节点让第一个节点和后续节点的处理方式完全一样，不需要单独判断"新链是否为空"。

## [合并 K 个升序链表](https://leetcode.cn/problems/merge-k-sorted-lists/)（LeetCode 23）

### 问题

给你 K 个升序链表，把它们合并成一个升序链表。

### 思路

最直接的做法是顺序合并：先合并前两条，结果再和第三条合并……但这样每一轮都要把之前合并好的节点重新扫一遍，效率越来越差。

更好的做法是分治：K 条链表两两配对合并，这一轮结束后剩 K/2 条；再两两配对，剩 K/4 条……直到只剩一条。每个节点只被操作 log K 次。

```
第一轮：[l1, l2, l3, l4] → [merge(l1,l2), merge(l3,l4)]
第二轮：                  → [merge(结果1, 结果2)]
```

```go
func mergeKLists(lists []*ListNode) *ListNode {
    if len(lists) == 0 {
        return nil
    }
    return mergeRange(lists, 0, len(lists)-1)
}

func mergeRange(lists []*ListNode, left, right int) *ListNode {
    if left == right {
        return lists[left]
    }
    mid := (left + right) / 2
    l := mergeRange(lists, left, mid)
    r := mergeRange(lists, mid+1, right)
    return mergeTwoLists(l, r) // 直接复用 LeetCode 21 的代码
}
```

这道题本身没有新技术点。它的意义在于：把合并两个链表这个操作当成一块积木，通过分治组合出更大的能力。

## [排序链表](https://leetcode.cn/problems/sort-list/)（LeetCode 148）

### 问题

对一个链表做 O(n log n) 排序。

### 思路

链表没有下标，快排需要频繁跳跃访问，不适合。归并排序更适合链表：

1. 找到链表的中点，把链表切成两段
2. 对两段各自递归排序
3. 合并两段有序链表（就是 LeetCode 21）

找中点用快慢指针（和 [[02-双指针系列]] 里的方法一样），合并用 `mergeTwoLists`。

```go
func sortList(head *ListNode) *ListNode {
    if head == nil || head.Next == nil {
        return head
    }
    // fast 从 head.Next 出发，让 slow 落在左半段的末尾
    slow, fast := head, head.Next
    for fast != nil && fast.Next != nil {
        slow = slow.Next
        fast = fast.Next.Next
    }
    mid := slow.Next
    slow.Next = nil // 切断成两段

    l := sortList(head)
    r := sortList(mid)
    return mergeTwoLists(l, r)
}
```

`fast` 从 `head.Next` 而不是 `head` 出发，是为了保证 `slow` 最终落在左半段的末尾，让 `slow.Next` 能正确指向右半段的头。

这道题把两个已经学过的操作直接拼在一起——快慢指针找中点、`mergeTwoLists` 合并。如果前面两个都熟了，这道题写起来几乎没有障碍。

---

## 23 和 148 的共同点

两道题都把 `mergeTwoLists` 当作核心操作：

- 23 用分治把"合并两个"变成"合并 K 个"
- 148 用递归把"合并两个"变成"排序整条链表"

同一个操作，套不同的框架，解决不同规模的问题。
