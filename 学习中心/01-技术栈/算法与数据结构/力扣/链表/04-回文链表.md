# [回文链表](https://leetcode.cn/problems/palindrome-linked-list/)：把前面学的东西拼在一起

> 这道题是整个链表专题的综合题。它本身不难，但它把快慢指针和反转链表都用上了，读完前面几篇再来看，会有一种"刚好能用上"的感觉。

---

## 问题

判断一个链表是否是回文链表，要求 O(n) 时间、O(1) 额外空间。

```
1 → 2 → 2 → 1   → true
1 → 2 → 3       → false
```

**O(1) 空间**这个约束很关键，它堵死了"把所有值复制到数组，再双指针判断"的思路。必须在链表结构本身上做文章。

---

## 怎么想到解法

回文的直觉是：前半段和后半段镜像对称。

如果能把后半段**反转**，再和前半段逐个比较，就能判断是否回文。

所以问题变成三步：
1. 找到链表的中点，把链表分成前后两半
2. 反转后半段
3. 逐节点比较前后两半

步骤 1 用快慢指针（来自[[02-双指针系列]]），步骤 2 复用反转链表（来自[[01-反转系列]]），步骤 3 最简单。

---

## 步骤一：快慢指针找中点

fast 每次走 2 步，slow 每次走 1 步，fast 到末尾时 slow 在中点。

```go
slow, fast := head, head
for fast.Next != nil && fast.Next.Next != nil {
    slow = slow.Next
    fast = fast.Next.Next
}
// slow 停在中点（奇数长度）或左中点（偶数长度）
// slow.Next 是后半段的起点
```

这里用 `fast.Next != nil && fast.Next.Next != nil` 而不是 `fast != nil && fast.Next != nil`，两者的差别是：前者让 slow 停在中点的前一个位置，后者让 slow 停在中点本身。我们想通过 `slow.Next` 拿到后半段的头，所以用前者。

---

## 步骤二：反转后半段

直接复用反转链表函数：

```go
secondHalf := reverseList(slow.Next)
```

反转后，`secondHalf` 指向后半段的新头节点（原来的最后一个节点）。

---

## 步骤三：逐节点对比

```go
p1, p2 := head, secondHalf
result := true
for p2 != nil {
    if p1.Val != p2.Val {
        result = false
        break
    }
    p1 = p1.Next
    p2 = p2.Next
}
```

以 `p2`（后半段）为循环终止条件：奇数长度时，前半段比后半段多一个节点（中点），用 `p2` 来控制循环可以自动跳过前半段多出来的中点。

---

## 完整代码

```go
func isPalindrome(head *ListNode) bool {
    if head == nil || head.Next == nil {
        return true
    }

    // 找中点
    slow, fast := head, head
    for fast.Next != nil && fast.Next.Next != nil {
        slow = slow.Next
        fast = fast.Next.Next
    }

    // 反转后半段
    secondHalf := reverseList(slow.Next)

    // 对比
    p1, p2 := head, secondHalf
    result := true
    for p2 != nil {
        if p1.Val != p2.Val {
            result = false
            break
        }
        p1 = p1.Next
        p2 = p2.Next
    }

    // 恢复链表（如果不需要保持原结构可以省略）
    slow.Next = reverseList(secondHalf)

    return result
}

func reverseList(head *ListNode) *ListNode {
    var prev *ListNode
    curr := head
    for curr != nil {
        next := curr.Next
        curr.Next = prev
        prev = curr
        curr = next
    }
    return prev
}
```

## 这道题为什么放在最后

在解这道题之前，如果已经熟悉快慢指针和反转链表，整个过程就是：找中点（会）→ 反转后半（会）→ 对比（简单）。思路清晰，没有任何新技巧需要发明。

但如果没有那两个基础，就会觉得这道题很复杂，因为你要同时想三件事。

这说明了一个学习上的事实：**算法题的"复杂度"很大程度上取决于你对子技术的熟悉程度**。同一道题，对不同的人来说难度是不一样的。

---

## 回顾整个链表专题

这九道题翻来覆去就是在做三件事：

- **两个指针联动**：感知链表中"直接看不到"的位置（相交/环形/删除倒数/本题找中点）
- **改写 Next 方向**：在不开辟额外空间的情况下重组结构（反转系列/本题反转后半段）
- **虚拟头节点**：消除头节点没有前驱带来的边界特判（合并/两数相加/删除节点/交换节点）

链表的约束（只能顺序游走）逼出了这些技巧，而这些技巧放到别的数据结构上大多不适用——这正是链表作为一个独立学习专题的价值所在。
