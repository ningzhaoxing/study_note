# 链表算法：核心思维框架

> 这是所有链表题目的总览。每当解一道新题时，回来找它的位置。

---

## 链表的本质约束

链表和数组的区别只有一条：

> **数组可以直接用下标访问任意位置，链表只能从头一步一步往后走。**

但链表有一个数组没有的自由：**节点之间的连接关系可以被随时改写**。数组元素的物理位置是固定的，链表的"顺序"只存在于 `Next` 指针里，指针指向哪，链表就往哪延伸。

所以学链表算法，其实是在学：在"只能顺序游走"这个约束下，怎么通过操控指针解决各种问题。

这九道题翻来覆去，核心就是在做三件事：

**① 用两个指针同时感知不同的位置**（因为单个指针只能看到当前节点）
**② 修改 `Next` 指针来重组结构**（这是链表区别于数组的核心能力）
**③ 在头节点前面加一个虚拟头节点，消除边界特判**

每道题都是这三件事的不同组合，后面会一直往这里靠。

## Go 链表节点定义

```go
type ListNode struct {
    Val  int
    Next *ListNode
}
```

链表的全部复杂性，都来自对这个简单结构的操控。
