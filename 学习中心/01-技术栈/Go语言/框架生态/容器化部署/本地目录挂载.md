基于宿主机目录，实现容器目录挂载。

# 实现本地目录挂载
- 在执行`docker run`命令时，使用`-v 本地目录:容器内目录`可以完成本地目录挂载。
- 本地目录必须以`/`或`./`开头。直接以名称开头，会被识别未数据卷而非本地目录。
	- `-v mysql:/var/lib/mysql` 被识别为一个数据卷叫mysql
	- `-v ./mysql:/var/lib/mysql`被识别为当前目录的mysql目录

# 挂载本地目录到mysql容器目录
## 1. 挂载data目录
挂载`/root/mysql/data`到容器内的`/var/lib/mysql`目录。

`-v /root/mysql/data:/var/lib/mysql`

## 2. 挂载initdb目录
挂载`/root/mysql/init`到容器内的`/docker-entrypoint-initdb.d`目录。 # SQL执行脚本

`-v /root/mysql/init:/docker-entrypoint-initdb.d`

## 3. 挂载conf目录
挂载`/root/mysql/conf`到容器内的`/ect/mysql/conf.d`目录。 # 配置文件

`-v /root/mysql/conf:/ect/mysql/conf.d`

因此，mysql容器启动命令应如下：
```Docker
docker run -d \
	--name mysql \
	-p 3306:3306 \
	-e TZ-Asia/Shanghai \
	-e MYSQL_ROOT_PASSWORD=xxx \
	-v /root/mysql/data:/var/lib/mysql \
	-v /root/mysql/init:/docker-entrypoint-initdb.d \
	-v /root/conf:/ect/mysql/conf.d \
mysql	
```

> 注意：执行命令前，需要将不存在的本地目录提前创建好。

# 挂载本地目录到nginx容器目录

```linux
docker run -d \
	--name nginx \
	-p 18080:18080 \
	-p 18081:18081 \
	-v /root/nginx/html:/usr/share/nginx/html \
	-v /root/nginx/nginx.conf:/etc/nginx/nginx.conf \
	--network 网络名 \
nginx
```
