# 反转系列：改写 Next 方向的三道题

> 这三道题做的是同一件事，只是规模不同：反转整条链表、每次反转 2 个、每次反转 K 个。

---

## [反转链表](https://leetcode.cn/problems/reverse-linked-list/)（LeetCode 206）

### 问题

给你单链表的头节点，反转整条链表并返回新的头节点。

### 核心动作

反转，就是把每个节点的 `Next` 从"向后指"改成"向前指"。

```
原来：  1 → 2 → 3 → nil
反转后：nil ← 1 ← 2 ← 3
```

问题在于：当你把 `curr.Next` 改成 `prev` 的时候，原来的 `curr.Next` 就丢了——你再也找不到链表的下一个节点了。

所以改之前必须先把下一个节点存起来：

```go
next := curr.Next   // 先保存，不然改完就找不到了
curr.Next = prev    // 改方向
prev = curr
curr = next
```

这四行的顺序不能动，其余的都是让 `prev` 和 `curr` 往前推进。

```go
func reverseList(head *ListNode) *ListNode {
    var prev *ListNode
    curr := head
    for curr != nil {
        next := curr.Next
        curr.Next = prev
        prev = curr
        curr = next
    }
    return prev // curr 走到 nil 时，prev 就是原来的最后一个节点，也就是新的头
}
```

### 递归写法

递归的思路是：先把 `head.Next` 之后的所有节点递归反转好，然后再处理 `head` 和它后面节点的连接。

```go
func reverseList(head *ListNode) *ListNode {
    if head == nil || head.Next == nil {
        return head
    }
    newHead := reverseList(head.Next)
    // 此时 head.Next 还指着原来第二个节点
    // 那个节点现在是反转后链表的尾部，让它指回 head
    head.Next.Next = head
    head.Next = nil // head 变成尾部，断开原来的连接
    return newHead
}
```

两种写法本质相同，都是逐个改方向。迭代从头开始往后，递归从尾开始往前。递归更难理解，但有时候能帮助思考"当前节点做什么"这个问题。

## [两两交换链表中的节点](https://leetcode.cn/problems/swap-nodes-in-pairs/)（LeetCode 24）

### 问题

每两个节点交换一次，返回头节点。
- `1 → 2 → 3 → 4` 变成 `2 → 1 → 4 → 3`

### 这道题本质上是 K=2 的 K 组反转

每次处理一对节点 `[a, b]`，要完成：
```
before:  prev → a → b → 后续
after:   prev → b → a → 后续
```

需要做三步连接：`prev → b`、`b → a`、`a → 后续`。

顺序很关键，必须先把 `b.Next`（即"后续"的头）保存下来，否则接完之后找不到后面的链表了。

```go
func swapPairs(head *ListNode) *ListNode {
    dummy := &ListNode{Next: head} // 虚拟头节点，让头节点也有前驱
    prev := dummy

    for prev.Next != nil && prev.Next.Next != nil {
        a := prev.Next
        b := prev.Next.Next

        prev.Next = b
        a.Next = b.Next
        b.Next = a

        prev = a // a 现在是这对的尾部，移过去准备处理下一对
    }
    return dummy.Next
}
```

**为什么需要虚拟头节点？** 交换第一对时，需要修改某个前驱节点的 `Next`。没有虚拟头节点，头节点没有前驱，需要单独处理。加了虚拟头节点，所有节点都有前驱，代码逻辑上就会统一。

## [K 个一组翻转链表](https://leetcode.cn/problems/reverse-nodes-in-k-group/)（LeetCode 25）

### 问题

每 K 个节点为一组进行翻转，最后不足 K 个的保持原样。
- `1→2→3→4→5`，K=3 → `3→2→1→4→5`

### 思路

这道题没有新的技巧，它就是把"反转链表"这个操作重复用在每一组上，难点在于**每组反转完之后怎么接回去**。

每组反转前后，连接关系是这样变化的：
```
反转前：  groupPrev → [n1 → n2 → ... → nK] → groupNext
反转后：  groupPrev → [nK → ... → n2 → n1] → groupNext
```

反转后，`n1` 变成了这组的尾部，`nK` 变成了头部。需要做两个连接：
- `groupPrev.Next = nK`（前段接新头部）
- `n1.Next = groupNext`（新尾部接后段）

所以反转前要把 `groupNext` 记住，反转后 `n1` 还在原位，直接用。

```go
func reverseKGroup(head *ListNode, k int) *ListNode {
    dummy := &ListNode{Next: head}
    groupPrev := dummy

    for {
        // 找到这组的第 K 个节点，不足 K 个就停止
        kth := getKth(groupPrev, k)
        if kth == nil {
            break
        }
        groupNext := kth.Next // 记住下一组的起点

        // 反转 groupPrev.Next 到 kth 这一段
        // 让 prev 从 groupNext 开始，这样反转完 nK 的 Next 自然指向 groupNext
        prev, curr := groupNext, groupPrev.Next
        for curr != groupNext {
            next := curr.Next
            curr.Next = prev
            prev = curr
            curr = next
        }

        // 接回去
        tmp := groupPrev.Next // 反转后 groupPrev.Next 还指着原来的 n1，也就是新的尾部
        groupPrev.Next = kth  // 前段接上新的头部
        groupPrev = tmp       // 移动到下一组的前驱位置
    }
    return dummy.Next
}

func getKth(curr *ListNode, k int) *ListNode {
    for curr != nil && k > 0 {
        curr = curr.Next
        k--
    }
    return curr
}
```

`prev` 从 `groupNext` 开始是个技巧：反转的时候每个节点的 `Next` 都会指向它的前一个，当反转到 `nK` 时，`nK.Next` 自然就是 `groupNext`，不需要再手动连接。

## 三题的关系

```
反转链表（206）
    ↓ 每次只反转 2 个节点
两两交换（24）— 是 K=2 的特殊情况
    ↓ 把 2 推广到任意 K
K 个一组翻转（25）— 把反转封装后循环调用
```

核心动作始终是那四行：保存 `next`、改方向、推进 `prev`、推进 `curr`。题目越复杂，只是在不同的范围内重复这个动作。
