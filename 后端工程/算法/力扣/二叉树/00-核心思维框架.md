# 二叉树算法：核心思维框架

> 这是所有二叉树题目的总览。每当解一道新题时，回来找它的位置。

---

## 从链表到二叉树：一个分叉引发的变化

学链表时，我们知道链表的本质约束只有一条：**只能从头一步一步往后走**，但可以随意改写 `Next` 指针。

二叉树是链表的延伸——每个节点多了一个指针方向：

```go
// 链表节点：一个方向
type ListNode struct {
    Val  int
    Next *ListNode
}

// 二叉树节点：两个方向
type TreeNode struct {
    Val   int
    Left  *TreeNode
    Right *TreeNode
}
```

只是多了一个分叉，但带来了根本性的变化：**你再也不能用简单的线性游走处理整棵树了**。链表可以 `for curr != nil { curr = curr.Next }` 处理完所有节点，但树节点有两个方向，走 `Left` 就无法同时走 `Right`。

这个分叉，天然地呼唤出了递归。

---

## 聚焦点：递归的形状和树的形状是同构的

树在结构上是自我定义的——一棵树要么是空的，要么是"一个根节点 + 一棵左子树 + 一棵右子树"。因此，解树的算法的递归形状和这个定义完全一致：

```
solve(root) {
    solve(root.Left)   // 把问题交给左子树
    solve(root.Right)  // 把问题交给右子树
}
```

写树的递归，只需要想清楚三件事：

1. **这个函数是干什么的**（它对一棵子树，承诺返回什么）
2. **终止条件**（碰到 `nil` 时怎么处理）
3. **当前节点做什么**（如何利用子树返回的结果，或者在进入子树之前先做什么）

想清楚这三件事，递归会自动处理剩余的细节。这是所有树的题能被解决的根本原因。

## 前序、中序、后序：信息流动的方向

这三种遍历方式的区别，只有一点：**在什么时刻处理当前节点**。

```go
func traverse(root *TreeNode) {
    if root == nil { return }

    // ← 前序位置：进入这个节点时立刻处理
    traverse(root.Left)
    // ← 中序位置：左子树完成后再处理
    traverse(root.Right)
    // ← 后序位置：左右子树都完成后再处理
}
```

这不只是遍历顺序，更是信息流动的方向：

- **前序（自上而下）**：当前节点先"说话"，再传给子树。适合把信息从根向叶子传递，参数往下传。
- **后序（自下而上）**：子树先"说话"，当前节点再综合结果。适合从叶子向根汇聚信息，返回值向上传。
- **中序（左→根→右）**：对二叉搜索树有特殊意义——BST 的中序遍历恰好是升序序列。

后续大多数题都是在利用这两个方向中的一种。

---

## 两种思维模式

**模式一：自上而下（前序为主）**

当前节点把需要的信息向下传递给子节点，子节点继续处理。当前节点做判断，信息从根流向叶子。

```go
// 验证 BST：把合法范围向下传
func validate(node *TreeNode, min, max int) bool {
    if node == nil { return true }
    if node.Val <= min || node.Val >= max { return false }
    return validate(node.Left, min, node.Val) &&
           validate(node.Right, node.Val, max)
}
```

**模式二：自下而上（后序为主）**

先等子树把结果算出来，再在当前节点综合。信息从叶子流向根。

```go
// 最大深度：子树先汇报高度，当前节点取最大值加一
func maxDepth(root *TreeNode) int {
    if root == nil { return 0 }
    left := maxDepth(root.Left)
    right := maxDepth(root.Right)
    return max(left, right) + 1
}
```

大多数树的题用后序思维，因为树的答案往往要从叶子向根汇聚。但有时候同一道题可以用两种方式都写，区别在于信息是通过参数往下传还是通过返回值往上传。

---

## DFS vs BFS

树的遍历有两种大方向：

- **DFS（深度优先）**：一条路走到底，再回头。对应前/中/后序遍历，用递归（隐式调用栈）或显式栈实现。
- **BFS（广度优先）**：一层一层向外扩展。对应层序遍历，用队列实现。

DFS 自然地对应递归结构，BFS 自然地对应队列。需要"层"这个概念时（某一层的节点、每层最大值等）用 BFS；绝大多数树的问题用 DFS 更简洁。

```go
type TreeNode struct {

    Left  *TreeNode
    Right *TreeNode
}
```

树的全部复杂性，都来自对这个简单结构的递归操控。
