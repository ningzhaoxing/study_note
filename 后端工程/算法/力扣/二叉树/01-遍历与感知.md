# 遍历与感知：读取树的结构信息

> 这几道题不改变树的形状，只是从树里"读出"信息。它们是理解树的递归框架最好的入口。

---

## 前/中/后序遍历（LeetCode 144 / 94 / 145）

### 递归：最直接的表达

递归遍历几乎是"把树的定义直接翻译成代码"：

```go
func preorder(root *TreeNode, result *[]int) {
    if root == nil { return }
    *result = append(*result, root.Val) // 前序：进入节点立刻处理
    preorder(root.Left, result)
    preorder(root.Right, result)
}

func inorder(root *TreeNode, result *[]int) {
    if root == nil { return }
    inorder(root.Left, result)
    *result = append(*result, root.Val) // 中序：左子树完了再处理
    inorder(root.Right, result)
}

func postorder(root *TreeNode, result *[]int) {
    if root == nil { return }
    postorder(root.Left, result)
    postorder(root.Right, result)
    *result = append(*result, root.Val) // 后序：子树都完了再处理
}
```

这三段代码唯一的差别，就是那一行 `append` 放在哪里。这不是细节，而是**框架本身**：在哪里放代码，决定了信息流动的方向。

### 迭代：用显式栈模拟递归

递归之所以能"记住"回去的路，是因为系统帮你维护了调用栈。迭代遍历就是把这个隐式调用栈变成显式的。

**前序迭代**：把节点压栈，弹出时处理，先压右再压左（这样左先弹出）：

```go
func preorderTraversal(root *TreeNode) []int {
    var result []int
    if root == nil { return result }
    stack := []*TreeNode{root}
    for len(stack) > 0 {
        node := stack[len(stack)-1]
        stack = stack[:len(stack)-1]
        result = append(result, node.Val)
        if node.Right != nil { stack = append(stack, node.Right) } // 右先压
        if node.Left != nil { stack = append(stack, node.Left) }   // 左后压，先弹出
    }
    return result
}
```

**中序迭代**：中序需要先走到最左边，再往回。指针 `curr` 一直往左走并压栈，走不动了就弹出、访问，然后转向右子树：

```go
func inorderTraversal(root *TreeNode) []int {
    var result []int
    stack := []*TreeNode{}
    curr := root
    for curr != nil || len(stack) > 0 {
        for curr != nil {          // 一直往左走，沿途压栈
            stack = append(stack, curr)
            curr = curr.Left
        }
        curr = stack[len(stack)-1] // 退无可退，弹出
        stack = stack[:len(stack)-1]
        result = append(result, curr.Val) // 访问
        curr = curr.Right                 // 转向右子树
    }
    return result
}
```

**后序迭代**：后序的顺序是"左→右→根"，逆过来是"根→右→左"，这正好是前序的"先右后左"版本，最后反转结果即可：

```go
func postorderTraversal(root *TreeNode) []int {
    var result []int
    if root == nil { return result }
    stack := []*TreeNode{root}
    for len(stack) > 0 {
        node := stack[len(stack)-1]
        stack = stack[:len(stack)-1]
        result = append(result, node.Val)
        if node.Left != nil { stack = append(stack, node.Left) }   // 左先压
        if node.Right != nil { stack = append(stack, node.Right) } // 右后压，先弹出
    }
    // result 此时是"根→右→左"，反转得"左→右→根"
    for i, j := 0, len(result)-1; i < j; i, j = i+1, j-1 {
        result[i], result[j] = result[j], result[i]
    }
    return result
}
```

迭代版本更繁琐，但它揭示了递归的本质：**调用栈就是帮你"记住回去的路"**。在树的算法里，优先用递归，清晰得多。

## 层序遍历（LeetCode 102）

### 问题

把二叉树按层输出，每层的节点值放在一个数组里。

### BFS：队列的自然对应

DFS（递归）沿着深度方向走，BFS 沿着广度方向走。BFS 需要一个数据结构来"记住当前层还没处理的节点"，这就是队列——先进先出，保证一层处理完才处理下一层。

关键在于**区分层的边界**：每次进入循环时，队列里存的恰好是当前层的所有节点。把当前队列大小记下来，只处理这么多个节点，剩下新压入的就是下一层的。

```go
func levelOrder(root *TreeNode) [][]int {
    if root == nil { return nil }
    result := [][]int{}
    queue := []*TreeNode{root}
    for len(queue) > 0 {
        size := len(queue) // 当前层的节点数，先记下来
        level := []int{}
        for i := 0; i < size; i++ {
            node := queue[i]
            level = append(level, node.Val)
            if node.Left != nil { queue = append(queue, node.Left) }
            if node.Right != nil { queue = append(queue, node.Right) }
        }
        result = append(result, level)
        queue = queue[size:] // 去掉已处理的这一层
    }
    return result
}
```

`size := len(queue)` 是这道题的核心动作——在扩展下一层之前，把当前层的大小快照下来，这样才能区分"当前层"和"下一层"。

## 二叉树的最大深度（LeetCode 104）

### 问题

给定一棵二叉树，返回它的最大深度（从根节点到最远叶子节点的最长路径的节点数）。

### 最干净的后序例子

这道题是后序思维的最简单体现：一棵树的最大深度 = max(左子树深度, 右子树深度) + 1。

当前节点不知道自己有多深，但它可以**问左右子树**，然后取最大值，加上自身这一层。

```go
func maxDepth(root *TreeNode) int {
    if root == nil { return 0 }       // 空树深度为 0
    left := maxDepth(root.Left)
    right := maxDepth(root.Right)
    if left > right {
        return left + 1
    }
    return right + 1
}
```

这道题值得反复看：它是"后序自下而上汇聚信息"最干净的形式。后面的直径、LCA 都是同样的思路，只是在"如何综合左右子树结果"这一步更复杂。

## 对称二叉树（LeetCode 101）

### 问题

判断一棵二叉树是否关于中轴对称。

```
对称：       不对称：
    1             1
   / \           / \
  2   2         2   2
 / \ / \         \   \
3  4 4  3         3   3
```

### 镜像方向的递归

直觉上，对称就是左子树和右子树互为镜像。但"互为镜像"不是"完全相同"——镜像要求：

- 左树的**左**和右树的**右**相同
- 左树的**右**和右树的**左**相同

这个交叉比较，意味着递归需要同时维护两个指针，往相反方向走：

```go
func isSymmetric(root *TreeNode) bool {
    return isMirror(root.Left, root.Right)
}

func isMirror(left, right *TreeNode) bool {
    if left == nil && right == nil { return true }
    if left == nil || right == nil { return false }
    return left.Val == right.Val &&
        isMirror(left.Left, right.Right) &&  // 左的左 vs 右的右
        isMirror(left.Right, right.Left)     // 左的右 vs 右的左
}
```

注意函数签名：`isMirror` 接收的是两个节点，这两个节点必须互为镜像。和最大深度那道题"对一棵子树做处理"不同，这里的递归是"对两棵子树同时做处理"。

这种"同时递归两棵树"的模式，在构造题（后面的 [[02-变形与构造]]）里还会用到。

## 二叉树的直径（LeetCode 543）

### 问题

找出二叉树中任意两个节点之间最长路径的**长度**（边的数量，不是节点数）。这条路径不一定要经过根节点。

```
        1
       / \
      2   3
     / \
    4   5

最长路径：4 → 2 → 1 → 3（长度 3）
或者：    4 → 2 → 5（长度 2）
答案是 3
```

### 后序的精华：从"深度"中顺手得出"直径"

这道题的关键洞察：**经过某个节点的最长路径 = 它的左子树深度 + 右子树深度**。

路径可以拐弯——从左子树最深处，经过当前节点，延伸到右子树最深处。

但问题在于，最长路径不一定经过根节点，可能经过任意一个节点。所以我们需要对每个节点都做这个计算，取最大值。

这里有个很漂亮的设计：计算深度本身就是后序遍历（子树先算深度，当前节点综合）。我们可以在计算深度的过程中，**顺手更新全局最大直径**：

```go
func diameterOfBinaryTree(root *TreeNode) int {
    maxDiam := 0
    var depth func(*TreeNode) int
    depth = func(node *TreeNode) int {
        if node == nil { return 0 }
        left := depth(node.Left)
        right := depth(node.Right)
        // 后序位置：子树的深度已知，更新经过当前节点的最长路径
        if left+right > maxDiam {
            maxDiam = left + right
        }
        // 返回当前节点的深度，给父节点用
        if left > right {
            return left + 1
        }
        return right + 1
    }
    depth(root)
    return maxDiam
}
```

`depth` 函数对外承诺"返回这棵子树的深度"，但同时在后序位置做了一个副作用——更新全局变量 `maxDiam`。这是一个很常见的模式：**函数返回值服务于父节点，全局变量收集跨层级的信息**。

在 [[02-变形与构造]] 里的最近公共祖先，用的也是同样的模式。

## 五道题的共同点

这五道题做的都是同一件事：**从树里读取信息**。

- 前/中/后序遍历：建立框架，理解"在哪里写代码"意味着什么
- 层序遍历：换个维度读取，BFS 的基本形式
- 最大深度：最简单的后序，子树汇报高度，当前节点综合
- 对称二叉树：同时递归两棵树，镜像方向推进
- 直径：在求深度的过程中，顺手收集全局最优值

读树的结构信息，基本就是这几种方式的组合。
