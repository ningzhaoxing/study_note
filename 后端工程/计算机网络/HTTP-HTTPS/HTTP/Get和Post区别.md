# GET 和 POST 的区别

> 这是最常见的 HTTP 面试题，核心在于理解两者的**语义**和**规范**上的区别，而非浏览器的实现细节。

## 一、核心语义差异

| 维度 | GET | POST |
|------|-----|------|
| **语义** | 获取资源 | 提交数据，对资源进行处理 |
| **安全性** | ✅ 安全（不修改服务器状态） | ❌ 不安全（会修改服务器状态） |
| **幂等性** | ✅ 幂等（多次请求结果相同） | ❌ 非幂等（多次请求可能产生不同结果） |
| **可缓存** | ✅ 可被缓存（默认会缓存） | ❌ 默认不缓存 |

> **安全**：不修改服务器状态，只是读取数据
> **幂等**：执行多次与执行一次的效果相同

---

## 二、数据传输方式

### GET：数据放在 URL 中
```http
GET /search?keyword=HTTP&page=1 HTTP/1.1
Host: www.example.com
```
- 参数拼接在 URL 的**查询字符串**中
- 会被保存在浏览器历史记录、日志文件中
- 对**敏感数据**不安全

### POST：数据放在请求体中
```http
POST /login HTTP/1.1
Host: www.example.com
Content-Type: application/json

{"username": "admin", "password": "123456"}
```
- 数据在**请求体（Body）** 中，不显示在 URL 里
- 相对更私密（但没有加密，仍可被抓包）

---

## 三、数据长度限制

- **GET**：URL 有长度限制（浏览器和服务器对 URL 长度有限制，通常约 **2KB**）
- **POST**：请求体理论上无大小限制（实际由服务器配置决定）

> ⚠️ 注意：这是浏览器和服务器的实现限制，**并非 HTTP 规范的要求**

---

## 四、编码类型

| | GET | POST |
|--|-----|------|
| **编码类型** | `application/x-www-form-urlencoded` | 多种（JSON、form-data、multipart 等） |
| **文件上传** | ❌ 不支持 | ✅ 支持（`multipart/form-data`） |

---

## 五、浏览器行为差异

| | GET | POST |
|--|-----|------|
| **回退/刷新** | 无副作用 | 会重新提交数据（浏览器提示） |
| **书签收藏** | ✅ 可以收藏 | ❌ 不能收藏 |
| **浏览器历史** | ✅ 参数保留在历史中 | ❌ 数据不保存 |
| **预加载/预取** | ✅ 可预加载 | ❌ 不会预加载 |

---

## 六、TCP 包发送

- **GET**：通常一次性发送（请求头和数据一起发送）
- **POST**：有些浏览器实现分两次发送：
  1. 先发请求头，等服务器返回 `100 Continue`
  2. 再发送请求体

> ⚠️ 这是部分浏览器的**实现细节**，并非 HTTP 规范要求

---

## 七、面试常见追问

### Q：GET 请求可以有 Body 吗？
**A：技术上可以，但不推荐。** HTTP 规范没有禁止 GET 带 Body，但大多数服务器和代理会忽略 GET 的 Body，且语义上 GET 就是获取资源，不应该有请求体。

### Q：POST 真的比 GET 安全吗？
**A：不是。** POST 只是数据不在 URL 中，但数据在请求体中，同样是**明文传输**，同样可以被抓包截获。真正的安全需要 **HTTPS** 加密。

### Q：什么时候用 GET，什么时候用 POST？
- **GET**：查询操作（获取列表、获取详情）
- **POST**：提交操作（登录、创建资源、上传文件）
- 遵循 **RESTful** 原则：GET=查询，POST=创建，PUT=更新，DELETE=删除
