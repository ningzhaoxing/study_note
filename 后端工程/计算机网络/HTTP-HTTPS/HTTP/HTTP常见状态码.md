# HTTP 常见状态码

HTTP 状态码由**三位数字**组成，第一位数字代表响应类别：

| 类别 | 范围 | 含义 |
|------|------|------|
| **1xx** | 100-199 | 信息性响应，请求已收到，继续处理 |
| **2xx** | 200-299 | 成功，请求被成功接收并处理 |
| **3xx** | 300-399 | 重定向，需要进一步操作才能完成请求 |
| **4xx** | 400-499 | 客户端错误，请求包含语法错误或无法完成 |
| **5xx** | 500-599 | 服务器错误，服务器未能完成合法的请求 |

---

## 2xx 成功

### 200 OK
- 最常见的成功状态码
- 请求已成功处理，响应体中包含请求的资源

### 201 Created
- 请求成功且服务器创建了新资源
- 通常是 `POST` 创建资源后返回，`Location` 头部指向新资源

### 204 No Content
- 请求成功，但**响应体为空**
- 常用于 `DELETE` 请求成功后，或 PUT 更新成功但无需返回内容

### 206 Partial Content
- 客户端通过 `Range` 头请求资源的**一部分**，服务器成功返回该部分
- 常见于**断点续传**和**视频流**播放

---

## 3xx 重定向

> 3xx 响应通常包含 `Location` 头，告知客户端新的资源位置

### 301 Moved Permanently（永久重定向）
- 资源已**永久**移到新 URL
- 浏览器会缓存这个重定向，下次直接访问新地址
- ⚠️ POST 请求重定向后，浏览器可能改为 GET 请求

### 302 Found（临时重定向）
- 资源**临时**移到新 URL
- 浏览器不会缓存，每次都请求原 URL
- ⚠️ 同样可能将 POST 改为 GET（规范上不应该，但浏览器行为不一致）

### 303 See Other
- 明确要求客户端**使用 GET 请求**访问另一个 URL
- 常用于 POST 表单提交后重定向到结果页（PRG 模式）

### 304 Not Modified
- 客户端发送**条件请求**（如 `If-Modified-Since`），资源未发生变化
- 告知客户端使用**缓存**，响应体为空
- ⚠️ 注意：304 与重定向无关，是缓存命中的标志

### 307 Temporary Redirect（临时重定向）
- 与 302 类似，但**严格保持请求方法不变**（POST 还是 POST）
- 遵照 HTTP/1.1 规范

### 308 Permanent Redirect（永久重定向）
- 与 301 类似，但**严格保持请求方法不变**
- 遵照 HTTP/1.1 规范

> **301 vs 302 vs 307 vs 308 总结：**
>
> | | 永久 | 临时 |
> |--|------|------|
> | **可能改变方法** | 301 | 302/303 |
> | **严格保持方法** | 308 | 307 |

## 4xx 客户端错误

### 400 Bad Request（请求错误）
- 请求报文存在**语法错误**，服务器无法理解
- 如 JSON 格式错误、参数类型不对等

### 401 Unauthorized（未认证）
- 请求需要**身份认证**，且认证未通过或未提供
- 响应中包含 `WWW-Authenticate` 头，告知认证方式

### 403 Forbidden（禁止访问）
- 服务器理解请求但**拒绝执行**
- 与 401 的区别：401 是没有认证，403 是认证了但没有权限

### 404 Not Found（未找到）
- 服务器找不到请求的资源
- 也可用于不想透露资源是否存在的场景（如代替 403）

### 405 Method Not Allowed（方法不允许）
- 请求使用的 HTTP 方法被禁止
- 响应中包含 `Allow` 头，列出允许的方法

### 409 Conflict（冲突）
- 请求与资源的当前状态存在冲突
- 如并发修改同一资源时版本冲突

### 422 Unprocessable Entity（无法处理的实体）
- 请求格式正确，但语义上有错误（如字段验证失败）
- RESTful API 中常用

### 429 Too Many Requests（请求过多）
- 客户端在一定时间内发送了过多请求（限流）
- 响应中通常包含 `Retry-After` 头

---

## 5xx 服务器错误

### 500 Internal Server Error（服务器内部错误）
- 服务器遇到了**未知的错误**，无法完成请求
- 通常是代码 Bug 或意外异常

### 502 Bad Gateway（网关错误）
- 作为网关的服务器从上游服务器收到了**无效响应**
- 常见于 Nginx 反代后端服务崩溃

### 503 Service Unavailable（服务不可用）
- 服务器**暂时无法处理请求**（过载或维护中）
- 响应中可包含 `Retry-After` 头

### 504 Gateway Timeout（网关超时）
- 网关等待上游服务器响应**超时**
- 与 502 的区别：502 是收到无效响应，504 是没有收到响应

## 面试速记

```
200 OK         — 请求成功
204 No Content — 成功，无响应体
206 Partial    — 范围请求成功（断点续传）
301            — 永久重定向（缓存）
302            — 临时重定向（不缓存）
304 Not Modified — 缓存未过期
400 Bad Request  — 语法错误
401 Unauthorized — 未认证
403 Forbidden    — 无权限
404 Not Found    — 资源不存在
500 Server Error — 服务器内部错误
502 Bad Gateway  — 网关接收到无效响应
503 Unavailable  — 服务不可用
504 Timeout      — 网关超时
```
