# 什么是 HTTP

> HTTP（HyperText Transfer Protocol，超文本传输协议）是一种在计算机世界里专门在「**两点**」之间「**传输**」文字、图片、音频、视频等「**超文本**」数据的「**约定和规范**」。

## 三个关键词拆解

| 关键词     | 含义                                                             |
| ------- | -------------------------------------------------------------- |
| **协议**  | 使用计算机能理解的语言，确立了计算机之间交流通信的规范，以及相关的各种控制和错误处理方式                   |
| **传输**  | HTTP 是双向协议，客户端请求、服务端响应；数据在两点之间传输，允许中间有**中转或接力**（A → C → E → B） |
| **超文本** | 文字、图片、视频的混合体，且含有超链接，能从一个超文本跳转到另一个超文本                           |

## HTTP 的特点

### ✅ 无连接
- 每次连接只处理一个请求，服务器处理完客户端的请求并收到应答后即断开连接
- **HTTP/1.1 引入了持久连接（Keep-Alive）**来解决这个问题

### ✅ 无状态
- HTTP 协议对事务处理没有记忆能力，不能记住之前的请求
- **解决方案**：Cookie / Session / Token

### ✅ 基于 TCP/IP
- HTTP 使用 TCP 作为传输层协议，保证数据的可靠传输
- 默认端口：**HTTP → 80**，**HTTPS → 443**

### ✅ 应用层协议
- HTTP 位于 OSI 七层模型的**应用层**，处于 TCP/IP 模型的最顶层

## HTTP 工作原理

```
客户端                        服务端
  │                              │
  │── TCP 三次握手 ──────────────>│
  │                              │
  │── HTTP 请求报文 ─────────────>│
  │   GET /index.html HTTP/1.1   │
  │   Host: example.com          │
  │                              │
  │<─ HTTP 响应报文 ──────────────│
  │   HTTP/1.1 200 OK            │
  │   Content-Type: text/html    │
  │   <html>...</html>           │
  │                              │
  │── TCP 四次挥手 ──────────────>│
```

## HTTP 的缺点

### 🔴 不安全
- **明文传输**：报文内容可以被任何人读取
- **不验证身份**：无法确认通信双方的真实身份（可能遭受中间人攻击）
- **无法保证完整性**：报文可能在传输中被篡改

> 解决方案：**HTTPS** = HTTP + SSL/TLS 加密层

### 🔴 无状态
- 服务器不保存客户端状态，每次请求都是全新的

> 解决方案：**Cookie** + **Session**；或使用 **JWT Token**

### 🔴 队头阻塞（HTTP/1.1）
- 同一个连接中，前一个请求未响应，后续请求必须等待

> 解决方案：**HTTP/2** 的多路复用
