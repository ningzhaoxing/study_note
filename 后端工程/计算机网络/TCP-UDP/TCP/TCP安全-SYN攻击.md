# TCP 安全 — SYN 攻击

---

## 一、半连接队列与全连接队列

服务端在 TCP 三次握手过程中维护两个队列：

```
客户端 ──SYN──▶ [半连接队列 (SYN Queue)] ──ACK完成──▶ [全连接队列 (Accept Queue)]
                                                              │
                                                        accept() 取走连接
```

### 半连接队列（SYN Queue）

- 服务端收到客户端的 **SYN** 后，将连接放入半连接队列，状态为 `SYN_RCVD`
- 服务端回复 **SYN+ACK**，等待客户端的第三次握手 ACK
- **队列大小**由 `tcp_max_syn_backlog` 参数控制（Linux 默认 256 或 512）

### 全连接队列（Accept Queue）

- 服务端收到客户端的 **ACK**（第三次握手完成）后，连接从半连接队列移入全连接队列，状态变为 `ESTABLISHED`
- 应用程序通过 `accept()` 系统调用从全连接队列取走连接
- **队列大小** = min(backlog, `net.core.somaxconn`)，`listen(fd, backlog)` 中的 backlog 指定

### 队列满了怎么办？

| 队列         | 满了之后的行为                                                          |
| ---------- | ---------------------------------------------------------------- |
| **半连接队列满** | 直接丢弃新的 SYN 报文（或开启 SYN Cookie 绕过）                                 |
| **全连接队列满** | 根据 `tcp_abort_on_overflow` 决定：0 = 丢弃 ACK（让客户端重试），1 = 发送 RST 拒绝连接 |

## 二、SYN Flood 攻击

### 攻击原理

攻击者利用**伪造源 IP 地址**，向服务端大量发送 SYN 报文：

1. 服务端收到 SYN，将其放入半连接队列，并回复 SYN+ACK
2. 由于源 IP 是伪造的，SYN+ACK 无法到达真实客户端，也就不会有第三次握手的 ACK
3. 服务端持续重传 SYN+ACK，半连接队列被大量无效连接占满
4. **正常连接的 SYN 无法进入队列，服务不可用**

```
攻击者 (伪造IP1) ──SYN──▶ 服务端
攻击者 (伪造IP2) ──SYN──▶ 服务端   [半连接队列被打满]
攻击者 (伪造IP3) ──SYN──▶ 服务端
正常用户         ──SYN──▶ 服务端 ✗ (队列已满，被丢弃)
```

## 三、防御：SYN Cookie

### 核心思路

不使用半连接队列存储状态，而是将连接信息**编码进序列号**，收到第三次握手 ACK 时再验证。

### 流程

1. 服务端收到 SYN 后，**不分配半连接队列资源**
2. 根据 `时间戳 + IP + 端口 + 密钥` 计算哈希值，作为 SYN+ACK 的**序列号（ISN）** 发出
3. 客户端回复 ACK 时，ACK 号 = ISN + 1
4. 服务端通过**反算哈希**验证 ACK 号的合法性，验证通过才建立连接

### 效果

- 攻击者因无法收到 SYN+ACK，也就无法伪造合法的 ACK 号
- 服务端**不需要存储半连接状态**，半连接队列不会被占满
- 代价：SYN Cookie 不支持 TCP 选项（如 SACK、窗口扩大因子），性能略有损耗

### Linux 开启方式

```bash
# 查看 SYN Cookie 状态（1 = 开启）
sysctl net.ipv4.tcp_syncookies

# 开启 SYN Cookie
sysctl -w net.ipv4.tcp_syncookies=1
```

## 四、其他防御措施

| 措施 | 说明 |
|------|------|
| **增大半连接队列** | 调大 `tcp_max_syn_backlog`，提高攻击门槛 |
| **减少 SYN+ACK 重传次数** | 调小 `tcp_synack_retries`（默认 5），尽早释放无效半连接 |
| **防火墙限速** | 对单 IP 的 SYN 报文频率进行限制 |
| **IP 黑名单/清洗** | 识别并过滤攻击 IP |
