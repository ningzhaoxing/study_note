## 一、Sheet 1：系统建模（原"数据流图"）

### 1.1 问题

| 编号 | 问题 | 当前代码位置 |

| DFD-001 | Sheet 名称为"数据流图"，应改为"系统建模" | `standard_utils.go:179` |

### 1.2 改动方案

#### 改动1：重命名 Sheet（DFD-001）

文件：`tara/service/page/report/standard_utils.go`

```go
// 修改前（第179行）
sheetName := "数据流图"

// 修改后
sheetName := "系统建模"
```

> 注意：模板文件 `tara_template.xlsx` 中对应 Sheet 也需同步重命名，否则 `sheetExists` 检查会失败。

#### 改动2：修正表头字段名（DFD-003）

文件：`tara/service/page/report/standard_utils.go`

```go
// 修改前（第221行）
headers := []string{"通道ID", "通道名称", "数据流ID", "数据流名称", "数据流描述", "数据流关联的数据"}

// 修改后
headers := []string{"通道ID", "通道名称", "数据流ID", "数据流名称", "数据流描述"}
```

> 原第6列"数据流关联的数据"删除，第5列"数据流描述"保留（原字段名已正确，只需删除多余列）。

#### 改动3：新增合并单元格标题行（DFD-003）

在 `fillDataflowTable` 函数中，写表头之前插入一行合并单元格标题：

```go
// 在 startRow 行之前插入标题行（startRow 原为2，标题占第2行，表头移至第3行）
titleRow := startRow         // 第2行：合并单元格标题

dataStartRow := startRow + 2 // 第4行起：数据

// 写合并标题

// 设置标题样式（居中、加粗）
```

## 二、Sheet 2：损害场景

### 2.1 问题

| 编号 | 问题 | 当前代码位置 |
| --- | --- | --- |
| DS-001 | 缺少 Safety / Financial / Operational / Privacy 四个影响维度字段 | `standard_excel.go:396-472` |
| DS-002 | 缺少"影响评级"汇总字段 | `standard_excel.go:396-472` |

### 2.2 数据来源确认

`DamageItem` 结构体（`tara/model/service/damage.go:18`）已包含所有需要字段：

```go
type DamageItem struct {
    DamageID    string  // 损害ID       → 列A（已有）
    Damage      string  // 损害描述     → 列E（已有）
    Safety      string  // 安全等级     → 列F（待补充）
    Financial   string  // 财务等级     → 列G（待补充）
    Operational string  // 操作等级     → 列H（待补充）
    Privacy     string  // 隐私等级     → 列I（待补充）
    Impact      string  // 影响等级     → 列J（待补充）
}
```

数据已存在，只需在导出时补充写入。

### 2.3 改动方案

文件：`tara/service/page/report/standard_excel.go`，函数 `fillTaraDamageScenarios`

在现有 A-E 列写入之后，追加 F-J 列：

```go
// 列F: 安全等级 (Safety)
if err := excelFile.SetCellValue(sheetName, fmt.Sprintf("F%d", row), damageItem.Safety); err != nil {
    return libError.NewError(constant.FAILED_TO_SET_CELL_VALUE, err)
}

// 列G: 财务等级 (Financial)
if err := excelFile.SetCellValue(sheetName, fmt.Sprintf("G%d", row), damageItem.Financial); err != nil {
    return libError.NewError(constant.FAILED_TO_SET_CELL_VALUE, err)
}

// 列H: 操作等级 (Operational)
if err := excelFile.SetCellValue(sheetName, fmt.Sprintf("H%d", row), damageItem.Operational); err != nil {
    return libError.NewError(constant.FAILED_TO_SET_CELL_VALUE, err)
}

// 列I: 隐私等级 (Privacy)
if err := excelFile.SetCellValue(sheetName, fmt.Sprintf("I%d", row), damageItem.Privacy); err != nil {
    return libError.NewError(constant.FAILED_TO_SET_CELL_VALUE, err)
}

// 列J: 影响评级（取四维度最高值）
if err := excelFile.SetCellValue(sheetName, fmt.Sprintf("J%d", row), damageItem.Impact); err != nil {
    return libError.NewError(constant.FAILED_TO_SET_CELL_VALUE, err)
}
```

模板文件 `tara_template.xlsx` 中"损害场景" Sheet 的表头需补充 F-J 列：

| 列 | 表头 |
| --- | --- |
| F | 安全等级(Safety) |
| G | 财务等级(Financial) |
| H | 操作等级(Operational) |
| I | 隐私等级(Privacy) |
| J | 影响评级 |

## 三、Sheet 3：影响程度评估（删除）

### 3.1 问题

该 Sheet 内容与优化后的"损害场景" Sheet 完全重复，PRD 要求删除。

### 3.2 改动方案

#### 方案A（推荐）：从模板文件中删除该 Sheet

直接在 `tara_template.xlsx` 中删除"影响程度评估" Sheet。
代码侧 `fillTaraImpactAssessment`（`standard_excel.go:474-484`）已有 `sheetExists` 检查，Sheet 不存在时自动跳过，无需改代码。

#### 方案B：代码侧主动删除

如果模板文件不方便修改，在 `fillTaraTemplate` 中并行任务完成后，主动删除该 Sheet：

```go
// 在 wg.Wait() 之后添加
if sheetExists(excelFile, "影响程度评估") {
    if err := excelFile.DeleteSheet("影响程度评估"); err != nil {
        logger.WarnWithCtx(ctx, libError.NewError(constant.ERR_CODE_INTERNAL_ERROR, err))
    }
}
```

> 推荐方案A，改模板更干净，不留无用代码。
