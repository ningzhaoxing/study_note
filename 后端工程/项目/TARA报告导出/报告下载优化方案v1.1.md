## 一、Sheet 4：威胁场景及攻击路径

### 1.1 问题根因

**问题1：枚举值与 PRD 不一致（TS-001）**

`fillThreatAttackPathRowV2`（L246）使用的是**数值转换函数**（`convertElapsedTimeValueToChinese` 等），将 `TotalCost` 中的 int 值转为中文，但转换结果与 PRD 要求的枚举文本不一致：

| 字段 | 当前转换函数 | 当前输出示例 | PRD 要求 |
|------|------------|------------|---------|
| 运行时间 | `convertElapsedTimeValueToChinese(int)` | `"不到一天"` | `"1天以内"` |
| 专业知识 | `convertExpertiseValueToChinese(int)` | `"无"` / `"一般"` | `"外行"` / `"熟练"` / `"专家"` / `"多位专家"` |
| 操作机会窗口 | `convertWindowValueToChinese(int)` | `"无"` | `"无限制"` |
| 设备 | `convertEquipmentValueToChinese(int)` | `"无"` | `"标准"` |

同文件 L340 还有另一版本的 `fillThreatAttackPathRowV2`，使用**字符串翻译函数**（`translateElapsedTime`、`translateExpertise` 等），这些函数查 `constant.MapAttackExpertiseToChinese` 等映射表，结果是正确的。两个版本并存，L246 版本（数值转换）被实际调用，导致输出错误。

**问题2：所有攻击路径都展开为行（TS-002）**

`fillTaraThreatScenariosAndAttackPaths`（L76）遍历 `treeGroup.AttackPaths` 的所有路径，每条生成一行。PRD 要求每个威胁场景只导出可行性等级最高的那条路径。

### 1.2 修改方案

**① 修复 `fillThreatAttackPathRowV2`（L246 版本）：改用字符串翻译函数**

L246 版本从 `attackPath.TotalCost`（int 字段）取值，改为从 `attackPath`（string 字段）取值，并调用已有的字符串翻译函数：

```go
// 修改前（L277-283）：使用数值转换
elapsedTime = s.convertElapsedTimeValueToChinese(attackPath.TotalCost.ElapsedTime)
expertise = s.convertExpertiseValueToChinese(attackPath.TotalCost.SpecialistExpertise)
knowledgeToe = s.convertKnowledgeValueToChinese(attackPath.TotalCost.KnowledgeOfTOE)
opportunityWindow = s.convertWindowValueToChinese(attackPath.TotalCost.WindowOfOpportunity)
equipment = s.convertEquipmentValueToChinese(attackPath.TotalCost.Equipment)
attackFeasibilityRating = attackPath.TotalCost.FeasibilityLevel

// 修改后：使用字符串翻译函数（与 L340 版本保持一致）
elapsedTime = s.translateElapsedTime(attackPath.ElapsedTime)
expertise = s.translateExpertise(attackPath.Expertise)
knowledgeToe = s.translateKnowledge(attackPath.KnowledgeToe)
opportunityWindow = s.translateWindow(attackPath.OpportunityWindow)
equipment = s.translateEquipment(attackPath.Equipment)
attackFeasibilityRating = s.translateAttackFeasibility(attackPath.AttackFeasibilityRating)
```

> 前提：需确认 `modelParam.AttackPathItemV2` 结构体上是否有 `ElapsedTime`、`Expertise` 等 string 字段。若只有 `TotalCost`，则需修正 `convertElapsedTimeValueToChinese` 等函数的输出文本与 PRD 对齐。

**② `constant/attack_path.go`：补全 `MapExpertiseToChinese` 的 key 格式**

当前 `MapExpertiseToChinese`（L30）的 key 是 `"<1 day"` 格式，但数据库存储的可能是 `"lessThanOneDay"` 等变体。需确认实际存储值后补全：

```go
var MapExpertiseToChinese = map[string]string{
    "<1 day":    "1天以内",
    "<1 week":   "1周以内",
    "<1 month":  "1个月以内",
    "<6 months": "6个月以内",
    ">6 months": "超过6个月",
    // 补全可能的变体（确认数据库实际值后添加）
}
```

**③ `fillTaraThreatScenariosAndAttackPaths`：只取最可行攻击路径**

在 `standard_tara.go` 中新增辅助函数，在 `fillTaraThreatScenariosAndAttackPaths` 的遍历逻辑中调用：

```go
// 新增：从攻击路径列表中选取可行性等级最高的路径
// 等级相同时取列表中靠前的那条（保持 API 返回顺序稳定性）
func pickBestAttackPath(paths []modelParam.AttackPathItemV2) *modelParam.AttackPathItemV2 {
    feasibilityOrder := map[string]int{
        "High": 4, "Medium": 3, "Low": 2, "Very Low": 1,
    }
    if len(paths) == 0 {
        return nil
    }
    best := &paths[0]
    for i := 1; i < len(paths); i++ {
        if feasibilityOrder[paths[i].AttackFeasibilityRating] > feasibilityOrder[best.AttackFeasibilityRating] {
            best = &paths[i]
        }
    }
    return best
}
```

修改 `fillTaraThreatScenariosAndAttackPaths` L76：

```go
// 修改前：遍历所有攻击路径
for _, attackPath := range treeGroup.AttackPaths {
    if err := s.fillThreatAttackPathRowV2(..., &attackPath); err != nil { ... }
    row++
}

// 修改后：只取最可行路径
bestPath := pickBestAttackPath(treeGroup.AttackPaths)
if bestPath != nil {
    if err := s.fillThreatAttackPathRowV2(..., bestPath); err != nil { ... }
    row++
}
// else: 无攻击路径时跳过该威胁场景（与 PRD 3.7.5 场景1 对齐）
```

## 二、Sheet 5：风险评估与处置

### 2.1 问题根因

**问题：列顺序与 PRD 不符，导致数据错位（RA-001/RA-002）**

`fillTaraRiskAssessmentAndTreatment`（L422）当前列映射：

| 列 | 当前写入内容 | PRD 3.5.4 要求 |
|----|------------|--------------|
| A | 威胁场景ID | 威胁场景ID ✓ |
| B | 资产名称 ❌ | 威胁场景描述 |
| C | 威胁场景描述 ❌ | 损害场景ID |
| D | 风险值 ❌ | 损害场景描述 |

| F | 攻击可行性评级 ❌ | 风险处置决策 |

| H | 影响等级 ❌ | 受保护的资产ID |

整体错位 1-2 列，且缺少损害场景ID、损害场景描述、受保护的资产ID 字段。

### 2.2 修改方案

**重写 `fillTaraRiskAssessmentAndTreatment` 的列填充逻辑**，按 PRD 3.5.4 重新对齐：

```go
// A列: 威胁场景ID（不变）
excelFile.SetCellValue(sheetName, fmt.Sprintf("A%d", row), riskData.ThreatID)

// B列: 威胁场景描述（原C列内容）
```

## 三、Sheet 6：网络安全目标汇总

### 3.1 问题根因

**问题1：编号格式错误（CG-002）**

`convertThreatIDToGoalID`（`standard_utils.go` L153）将 `THR-001` 转为 `CS-Goal-001`，PRD 要求格式为 `CS-GOAL-1`（全大写，无前导零）。

**问题2：无用户定义目标时导出占位数据（CG-001）**

`fillTaraNetworkSecurityGoalSummary`（L506）直接遍历 `RiskDataList`，对每条风险记录都生成一行目标，且当 `NetworkSecurityTarget` 为空时自动拼接占位文本（L541-542）。PRD 要求：无用户定义目标时显示空表格。

**问题3：缺少关联字段（PRD 3.6.3）**

PRD 要求表头包含：网络安全目标编号 / 资产ID / 资产名称 / 网络安全目标描述 / 关联威胁场景ID / 关联损害场景ID，当前缺少后两个字段。

### 3.2 修改方案

**① 修复 `convertThreatIDToGoalID`（`standard_utils.go` L153）**

```go
// 修改前
func (s *ReportService) convertThreatIDToGoalID(threatID string) string {
    if strings.HasPrefix(threatID, "THR-") {
        number := strings.TrimPrefix(threatID, "THR-")
        return fmt.Sprintf("CS-Goal-%s", number)
    }
    return fmt.Sprintf("CS-Goal-%s", threatID)
}

// 修改后：格式改为 CS-GOAL-{序号}，全大写，去除前导零
func (s *ReportService) convertThreatIDToGoalID(threatID string) string {
    re := regexp.MustCompile(`(\d+)$`)
    match := re.FindString(threatID)
    if match != "" {
        num, _ := strconv.Atoi(match)
        return fmt.Sprintf("CS-GOAL-%d", num)
    }
    return fmt.Sprintf("CS-GOAL-%s", strings.ToUpper(threatID))
}
```

需在文件顶部 import 中添加 `"regexp"` 和 `"strconv"`（若未引入）。

**② 修复 `fillTaraNetworkSecurityGoalSummary`：过滤占位数据，补全字段**

```go
func (s *ReportService) fillTaraNetworkSecurityGoalSummary(...) *libError.TaraError {
    // ...
    row := 2
    for _, riskData := range dataResult.RiskDataList {
        // 过滤：跳过没有用户定义网络安全目标的记录
        if riskData.NetworkSecurityTarget == "" {
            continue
        }

        // A列: 网络安全目标编号（格式修正为 CS-GOAL-{序号}）
        goalID := s.convertThreatIDToGoalID(riskData.ThreatID)
        excelFile.SetCellValue(sheetName, fmt.Sprintf("A%d", row), goalID)

        // B列: 资产ID
        excelFile.SetCellValue(sheetName, fmt.Sprintf("B%d", row), riskData.ComponentID)

        // C列: 资产名称
        excelFile.SetCellValue(sheetName, fmt.Sprintf("C%d", row), riskData.AssetName)

        // D列: 网络安全目标描述
        excelFile.SetCellValue(sheetName, fmt.Sprintf("D%d", row), riskData.NetworkSecurityTarget)

        // E列: 关联威胁场景ID（新增）
        excelFile.SetCellValue(sheetName, fmt.Sprintf("E%d", row), riskData.ThreatID)

        // F列: 关联损害场景ID（新增）
        damageID := ""
        if riskData.DamageData != nil {
            damageID = riskData.DamageData.DamageID
        }
        excelFile.SetCellValue(sheetName, fmt.Sprintf("F%d", row), damageID)

        row++
    }

    // 过滤后无数据：只保留表头，不写任何数据行（符合 PRD 要求）
    return nil
}
```
